# Project Context

## Projeto

Agenda Simples

## Objetivo

Construir uma plataforma SaaS de agendamento de serviços, com foco em aprendizado prático de Engenharia de Software, boas práticas de desenvolvimento Back-End, arquitetura, testes, documentação e cloud.

## Perfil do Projeto

Este projeto não tem como objetivo ser apenas um CRUD simples.

O objetivo é simular o desenvolvimento de um produto real, aplicando práticas próximas às utilizadas em times profissionais de engenharia de software.

## Usuário Principal

Prestadores de serviço ou empresas que precisam gerenciar horários, serviços, profissionais e clientes.

## Usuário Secundário

Clientes que desejam agendar serviços de forma simples.

## Regra Zero do Projeto

Nenhuma tecnologia será adicionada ao projeto sem antes existir um problema real que ela resolva.

## Forma de uso da IA

A IA deve atuar como mentor técnico, arquiteto e revisor de código.

A IA não deve ser usada como programador principal.

Antes de solicitar código, o desenvolvedor deve tentar entender o problema, propor uma solução e justificar suas decisões.
